问题与经验库

把容易混淆的点固定下来

不清楚代码在哪里运行 Linux 基本命令不熟 cron 语法不熟 测试环境与生产环境容易混淆
  • 先看 pwd，确认自己在哪个目录。
  • 先用 ls 确认文件存在，再执行脚本。
  • Python 脚本用 python3 文件名.py 运行。
  • 修改正式任务前，先执行 crontab -l。
  • 不确定时先退出，不要急着保存。
